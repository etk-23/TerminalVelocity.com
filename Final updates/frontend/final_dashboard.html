ls Command


The ls command is used to list all files and directories in the Linux terminal. When executed, it shows the contents of a directory, allowing users to see the names of file, directories, and their details.

I told my computer to ls the files... now it's listing all the reasons it's the best.

NORMAL OPERATION

Syntax: ls [option] [file/directory]

This will display the contents of the current directory. By default, 'ls' lists files and directories in alphabetical order.

COMMON OPTIONS

1. -l: displays detailed information about files and directories.
2. -a: represents all files Include hidden files and directories in the listing.
3. -t: sorts files and directories by their last modification time, displaying the most recently modified ones first.
4. -r: used to reverse the default order of listing.
5. -S: sorts files and directories by their sizes, listing the largest ones first.

cd Command


The cd command allows you to change directories in Linux, making it easier to navigate through the file system and manage your files efficiently.

I'm really good at cd-ing to the right place... I guess you could say I always navigate well.

MOVE TO THE SUB-DIRECTORY

Syntax: cd {subdirectory_name}

This will put the user in a subdirectory. For example, if you are in the directory /name, execution of 'cd my_name' will take you to /name/my_name.

MOVE TO THE PARENT-DIRECTORY

Syntax: cd ..

This will move the user up one directory. For example, if you are in the directory /name/my_name, execution of 'cd ..' will take you to /name.

cp Command


The cp command is used to copy files or directories from one location to another. It is an essential command in Linux for managing files and directories. To perform this command, the user needs to have read permissions for the source file and write permissions for the destination directory.

Copied files successfully—now I have backups and duplicates!

NORMAL OPERATION

Syntax: cp [options] [source] [destination]

Here, replace [source] with the file or directory you want to copy, and [destination] with the path where you want to copy it.

COMMON OPTIONS

1. -r or --recursive: This option is used to copy directories recursively.
2. -i or --interactive: Prompts you before overwriting an existing file in the destination.
3. -u or --update: Copies only when the source file is newer than the destination file, or if the destination file is missing.
4. -v or --verbose: Displays detailed information about the files being copied.

touch Command


The touch command is used to create an empty file or to update the access and modification times of an existing file. If the file doesn't exist, it will be created with zero size. If the file exists, its timestamp will be updated without modifying the content.

I tried using the touch command on my computer, but it didn't feel very touching.

NORMAL OPERATION

Syntax: touch [options] [file_name]

Here, replace [file_name] with the name of the file you want to create or update.

COMMON OPTIONS

1. -c or --no-create: Does not create a new file if the file doesn't exist.
2. -a: Changes only the access time of the file.
3. -m: Changes only the modification time of the file.
4. -t: Allows specifying a custom timestamp for the file in the format `[[CC]YY]MMDDhhmm[.ss]`.

find Command


The find command is used to search for files and directories in a directory hierarchy. It is a powerful tool that allows you to search by various criteria such as file name, type, size, modification date, and more.

I used the find command, but it turns out I was just findin' more problems!

NORMAL OPERATION

Syntax: find [path] [options] [expression]

Here, replace [path] with the directory to start the search, and [options] and [expression] with the conditions for the search (e.g., file name, size).

COMMON OPTIONS

1. -name: Searches for files by name.
2. -type: Searches for files of a specific type, such as `f` for regular files or `d` for directories.
3. -size: Finds files based on their size (e.g., `+100M` for files larger than 100MB).
4. -mtime: Searches for files based on their modification time (e.g., `-mtime -7` for files modified in the last 7 days).
5. -exec: Executes a command on the files that match the search criteria.

cat Command


The cat command is the simplest command to use when you want to see the contents of a particular file. The only issue is that it simply unloads the entire file to your terminal. If you want to navigate around a huge file, should use less command alternatively.

My cat refuses to get on the terminal. Guess she doesn't like the cat command.

NORMAL OPERATION

Syntax: cat [option] [file]

[option] represents various command-line options. [file] represents the name of the file to be processed.

COMMON OPTIONS

1. -n: introduces line numbers, making it convenient to identify and reference specific lines within the file.
2. -s: comes in handy when dealing with files containing repeated empty lines. It suppresses these repetitions, providing a cleaner output.
3. -E: used to highlight the end of each line.
4. -v: creates non-printing characters visible (except for tabs and line breaks).
5. -T: shows tabs as ^I.

sed Command


The sed command is a non-interactive text editor that allows you to perform basic text transformations on an input stream (a file or input from a pipe). It's often used for tasks like searching and replacing text, deleting lines, inserting new lines, and more.

I used sed to trim my lines, but it looks like I still need to make some more cuts.

NORMAL OPERATION

Syntax: sed [options] [command] [file]

[options]: Optional flags to modify sed behavior. [command]: The operation or transformation you want to perform (such as substitution, deletion, etc). [file]: The file or files on which you want to perform the operation. If no file is specified, sed will read from the standard input.

COMMON OPTIONS

1. -i: Edit files in place.
2. -n: Suppress automatic printing of pattern space.
3. -e: Allows you to add multiple commands.
4. -f: Specify a file with sed commands to execute.

awk Command


The awk command is a powerful text processing tool in Unix/Linux used for pattern scanning and processing. It allows you to manipulate data and generate reports by processing input line by line, which can be used for tasks like searching, extracting, and modifying text from files or streams.

Awk is like a Swiss army knife for text, making it easy to dissect, rearrange, and analyze data.

NORMAL OPERATION

Syntax: awk [options] '[pattern] {action}' [file]

[options]: Optional flags to modify awk behavior. [pattern]: The condition or pattern to match. [action]: The operation to perform on lines that match the pattern. [file]: The file or files to process. If no file is specified, `awk` reads from the standard input.

COMMON OPTIONS

1. -F: Specify a custom field separator (default is whitespace).
2. -v: Set variables to be used within awk.
3. -f: Read awk commands from a file.
4. -W: Enable or disable specific features.

cut Command


The cut command is used to extract sections from each line of a file or input data. It allows you to cut out columns, characters, or fields from a file and display the desired portion. This command is especially useful when you want to extract specific data from structured files, like CSV or tab-delimited files.

I tried to use the cut command on my life, but it just left me feeling cut off!

NORMAL OPERATION

Syntax: cut [options] [file]

[options] represents various command-line options, and [file] is the name of the file to be processed. If no file is provided, it will read from the standard input.

COMMON OPTIONS

1. -f: Specify which fields to cut (fields are typically separated by a delimiter).
2. -d: Specify the delimiter (default is tab).
3. -c: Cut based on character positions.
4. -s: Suppress lines that do not contain the delimiter.
